# Role Choice Card

## 1. 我选择的方向

我选择 **Dev Builder** 作为 Week 2 的主方向。

更具体的定位是：

> Web3 应用开发，偏全栈和链上数据方向。

## 2. 为什么选择 Dev

我具备计算机和数据科学背景，也学习过 Java、Spring Boot、MySQL 和数据分析。

在 Week 1 中，我已经完成了：

- 创建课程专用钱包
- 配置 Monad Testnet
- 完成测试网交易
- 编写 Solidity 合约
- 使用 Remix 编译和部署合约
- 调用合约函数
- 查询合约地址和交易哈希
- 使用 GitHub 整理项目记录

因此，与 Research 和 Ops 相比，Dev 更符合我的现有能力和职业发展方向。

我希望把原有的 Java 后端、数据库和数据分析能力，与 Solidity、React、钱包交互和链上数据结合起来。

## 3. Dev Builder 服务的问题

Dev Builder 主要负责把需求转化为可以运行和验证的产品。

在本项目中，我希望解决的问题是：

> 用户如何通过一个普通网页连接钱包，并在 Monad 链上完成每日打卡、查看签到记录和连续签到状态。

## 4. 本周最小可交付成果

Week 2 的最小产出是完成一个可以运行的 Monad Check-in DApp Demo，至少包括：

- React 前端页面
- MetaMask 钱包连接
- 显示当前钱包地址
- 连接 Monad Testnet
- 调用已部署的 Solidity 合约
- 完成一次链上打卡
- 查询用户打卡状态
- 保存交易哈希和运行截图
- 完善 GitHub README 和 Build Log

本周优先技术范围：

```text
React + MetaMask + Solidity + Monad
```
Spring Boot、MySQL 和数据看板作为后续扩展。
